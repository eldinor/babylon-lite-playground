Playground User Guide

This guide explains how to use the Babylon Lite Playground, write scenes with the public @babylonjs/lite API, upload assets, insert generated code, save snippets, and share examples.

Write normal module code Import from @babylonjs/lite just like in a local project.
Use the preview canvas The runner creates #renderCanvas before your scene runs.
Upload and insert assets Select an uploaded asset and press Insert to generate loader code.
Babylon Lite requires WebGPU. Use a current Chromium-based browser with WebGPU enabled. Browser support details are available at caniuse.com/webgpu.

Interface Overview

Area What It Does

Preview Sandboxed iframe with a fresh canvas for each run.

Write Scene Code

A scene is a browser ES module. It must export a zero-argument createScene function. The playground creates a canvas before importing your code. 

import {
  addToScene,
  attachControl,
  createArcRotateCamera,
  createBox,
  createEngine,
  createHemisphericLight,
  createSceneContext,
  createStandardMaterial,
  registerScene,
  startEngine,
} from "@babylonjs/lite";

export async function createScene() {
  const canvas = document.querySelector("#renderCanvas");
  const engine = await createEngine(canvas);
  const scene = createSceneContext(engine);

  const camera = createArcRotateCamera(Math.PI / 4, Math.PI / 3, 6, { x: 0, y: 0.8, z: 0 });
  attachControl(camera, canvas, scene);
  scene.camera = camera;
  scene.clearColor = { r: 0.04, g: 0.055, b: 0.075, a: 1 };

  addToScene(scene, createHemisphericLight([0, 1, 0], 0.8));

  const material = createStandardMaterial();
  material.diffuseColor = [0.2, 0.55, 1];

  const box = createBox(engine, 1.5);
  box.material = material;
  addToScene(scene, box);

  await registerScene(scene);
  await startEngine(engine);
}
The editor accepts TypeScript-style completions, but preview code runs as raw browser JavaScript. Do not use TypeScript-only syntax like api: any.

Use The Public Babylon Lite API

The playground uses an import map so snippets can import the package by name:

import * as Lite from "@babylonjs/lite";

or import only what you need:

import { createEngine, createSceneContext, loadGltf } from "@babylonjs/lite";

The playground does not expose a custom API object. If a function is publicly exported by the installed @babylonjs/lite package, import it normally.

Camera And Mouse Controls

Create an arc rotate camera, assign it to the scene, and attach pointer controls:

const camera = createArcRotateCamera(Math.PI / 4, Math.PI / 3, 6, { x: 0, y: 1, z: 0 });
attachControl(camera, canvas, scene);
scene.camera = camera;

Passing scene to attachControl is important. Babylon Lite applies orbit inertia from the scene render loop.

Upload And Insert Assets

Use the asset panel to upload files. Files are converted to data URLs for the sandboxed iframe. The original filename is preserved as a URL fragment, so .glb files are detected correctly.

  1. Click Upload and choose one or more files.
  2. Click an asset row to select it.
  3. Place your editor cursor inside createScene(), or let Insert find the scene setup line.
  4. Click Insert.

Insert looks for const scene = createSceneContext(...) or const scene = Lite.createSceneContext(...) and places generated code after that line.

Inserted Code By File Type

File Generated Code
.glb, .gltf Uses loadGltf(engine, url) and addToScene(scene, container).
Image files Uses loadTexture2D(engine, url).
Other files Inserts a URL lookup only.

Manual GLB Loading

Add the import at the top of the module:

import { addToScene, loadGltf } from "@babylonjs/lite";

Then place this inside createScene() after engine and scene are created:

const url = window.playgroundAssets["model.glb"];

if (url) {
  const container = await loadGltf(engine, url);
  addToScene(scene, container);
}

Uploaded assets are session-local and are not included in share URLs.

Saved Snippets

Snippets are stored in browser localStorage. No backend is used.

Control Behavior
Snippet select Switches between saved snippets.
Save Saves current editor source into the selected snippet.
Duplicate Creates a copy from the current editor source.
Delete Removes the selected snippet after confirmation.

Share URLs

The Share button compresses the current source with lz-string and stores it in the URL hash. This is good for small and medium snippets.

Uploaded assets are not included in share URLs. A recipient must upload the same files locally.

ZIP Export

Use Export ZIP when you want to save the current playground together with uploaded assets. This works without a database or backend.

The downloaded ZIP contains:

playground.js
share-url.txt
assets/
  uploaded-file.ext
File Contents
playground.js The plain source code currently in the editor.
share-url.txt The playground name, uploaded asset list, current share URL, and a note that uploaded assets are stored separately in the ZIP.
assets/ Every uploaded asset from the current browser session.

Use Import ZIP to restore an exported playground. The app reads playground.js, creates a new snippet using the name from share-url.txt, and loads every file from assets/ back into the asset panel.

Imported assets are still browser-session assets. Save another ZIP if you need to keep the restored playground and its files together again.

Console And Errors

The preview captures:

  • console.log
  • console.warn
  • console.error
  • window.error
  • window.unhandledrejection

Use the console panel to debug syntax errors, missing imports, missing assets, and runtime failures.

Troubleshooting

Mouse Controls Do Not React

Use attachControl(camera, canvas, scene), then assign scene.camera = camera.

ReferenceError: engine is not defined

Asset code was inserted outside the part of createScene() where engine exists. Insert usually places code after createSceneContext; otherwise move the generated block after your engine and scene setup.

SyntaxError: Unexpected token ':'

The preview runs JavaScript modules. Remove TypeScript annotations from runnable code.

WebGPU Is Unavailable

Use a current Chromium-based browser with WebGPU enabled.

Share URL Does Not Restore Assets

This is expected. Share URLs include source code only.
